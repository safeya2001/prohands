# Design Critique Report — Pro Hands Website Redesign
**Agent:** Design Critique Agent
**Date:** 2026-03-25
**Scope:** Full audit of `images/pro-hands.html` (live site archive) + `css/style.css` (new design system); deliverable is `index.html`.

---

## 1. Current Site Weaknesses Identified

### 1.1 Technical Architecture
- **WordPress/Elementor dependency.** The entire site is generated by Elementor's page-builder, producing deeply nested `div` soup with no semantic HTML. There are zero `<article>`, `<section>`, `<nav>`, `<header>`, or `<footer>` elements — only class-laden `<div>` and `<section>` wrappers that carry no meaning to assistive technology or search engines.
- **Massive CSS payload.** Sixteen separate external stylesheets are loaded (Elementor frontend, post-specific CSS, widget CSS, swiper, ekiticons, etc.), most of which serve content that does not appear on the homepage. This creates significant render-blocking overhead.
- **jQuery loaded synchronously** in `<head>`, blocking parse. The entire interaction layer depends on WordPress jQuery + jQuery Migrate — adding roughly 90 KB of blocking JavaScript for a primarily static marketing page.
- **No `<title>` brand.** The page title is simply `pro-hands` — no descriptive copy, no keyword signal, no brand positioning.
- **No OG/Twitter meta tags.** Social sharing produces unformatted link previews with no image or description.
- **No favicon in the new design path.** The favicons exist on the live WordPress CDN (`wp-content/uploads/`) but are not referenced with local paths in any local build.

### 1.2 Content & Hierarchy
- **Hidden content everywhere.** Key sections are marked `elementor-hidden-desktop elementor-hidden-tablet elementor-hidden-mobile` — meaning they are literally never rendered. This includes: the volunteer call-to-action section with the subscription form, the "Join Us Now!" section, the "Our Actions / For Animals" section, and the hero video. The site is visually thinner than its content inventory suggests.
- **Placeholder copy.** Multiple live sections contain Lorem Ipsum ("Lorem ipsum dolor sit amet…", "Maecenas at lorem…", "Suspendisse porttitor interdum…"), which undermines credibility.
- **Mission text mismatch.** The live Mission card reads "Empowering communities through education, creativity, and environmental awareness for a sustainable future" — this is weaker and less specific than the redesign spec, which provides the preferred copy "Educating communities through creative mediums like puppetry and theatre for environmental awareness."
- **No clear primary CTA in the hero.** The hero section has an h1 and h2 label ("Pro Hands") but no buttons, no tagline paragraph (the paragraph widget is hidden), and no directional link. The user lands on a full-screen background with a heading and immediately has to scroll to understand what to do.
- **"Our Core Values" section** is duplicated. The Values appear once in the Mission/Vision/Values 3-column grid and again in a separate "Fighting For Planet / Our Core Values" section with pie-chart widgets — creating structural redundancy.
- **No impact/statistics section visible.** The stat counter ("480 ton of trash every minute") is buried inside the About section as a small overlaid widget, not surfaced in a dedicated high-visibility section. This metric is powerful and should lead a section.

### 1.3 Visual Design
- **No visual identity system.** The site uses a generic Hello Elementor theme. Typography defaults to system fonts where Google Fonts fail to load. No consistent colour palette is enforced via CSS custom properties.
- **Inconsistent card treatment.** The Mission/Vision/Values section uses `elementor-image-box` widgets with PNG icons that have heavy drop shadows and mismatched sizes. The project cards follow the same pattern. Neither set uses hover states, 3D transforms, or visual feedback.
- **Section backgrounds clash.** The volunteer section and "Core Values" section both use near-identical dark overlays with insufficient contrast distinction between them, making the page feel repetitive in the lower half.
- **No scroll-reveal animations.** All content loads flat — there is no progressive reveal, parallax, or micro-interaction to guide the user's eye or communicate depth.
- **Globe/hero visual is absent.** The hero right column exists in the markup but its content widget is hidden on all breakpoints. The right half of the hero is empty whitespace.

### 1.4 Navigation
- **No sticky header.** The nav scrolls away immediately, requiring users to scroll all the way back to access other sections on a long-scrolling homepage.
- **No active state management.** Nav links have no active/current indicator (beyond an Elementor-added `active` class that carries no visible style in the base theme).
- **Hamburger closes but does not open reliably** in archived snapshot (relies on ekit JS that is not fully loaded in the local file).
- **No skip-to-content link.** Keyboard users and screen-reader users have no way to bypass the navigation.

## 2. Brand Alignment Assessment

### What the brand communicates
Pro Hands operates at the intersection of **environmental activism** and **creative education**. The brand name implies dexterity, craft, and community (hands that build, hands that puppeteer, hands that join). The organisation targets Jordanian communities — a bilingual (Arabic/English) audience — and positions itself as imaginative, grassroots, and purpose-driven.

### Current brand alignment: WEAK
| Dimension | Current Site | Expected |
|---|---|---|
| Colour palette | Undefined — inherits Elementor defaults (orange buttons, white bg) | Navy (`#2D3B55`), Teal (`#3DBFB8`), Yellow (`#F5C518`), Coral (`#E8445A`) |
| Typography | System sans-serif fallback; Google Font load is unreliable | Outfit (headings) + Lato (body) |
| Tone of voice | Generic ("Empowering communities…") with Lorem Ipsum contamination | Specific, action-oriented, community-first |
| Logo placement | Prominently placed in header — correct | Correct, to preserve |
| Imagery | Stock photography (woman with gas mask, rhino, red panda) that feels disconnected from Jordan-specific environmental work | Should prefer community/hands imagery; `group-of-environmental-conservation-people-hands-p-PXQ49EB-1.jpg` is available and on-brand |
| Iconography | Mixed: custom EkitIcons font, SVG inline icons, PNG pngwing imports — no system | Should standardise to inline SVG Heroicons-style set |

### New design brand alignment: STRONG
The new `index.html` and `style.css` establish a coherent design system:
- CSS custom properties lock in the full colour palette and cannot be accidentally overridden by third-party plugins.
- Outfit (geometric, modern, warm) communicates creative energy; Lato (humanist) ensures body text is readable and approachable.
- The navy + teal + yellow palette reads as trustworthy (navy), sustainable (teal/green-adjacent), and optimistic (yellow) — aligning with environmental NGO conventions while remaining distinctive.
- The 3D globe hero visual and floating particles communicate global environmental awareness without relying on stock photography.

## 3. Content Hierarchy Analysis

### Priority order — what users need to understand
1. **Who Pro Hands is and what they do** — must be answered above the fold (hero)
2. **Why it matters** — the environmental urgency (480 ton statistic, About section)
3. **What specifically they do** — Mission/Vision/Values + Projects
4. **How to get involved** — Volunteer section + Newsletter
5. **How to reach them** — Footer contact

### New `index.html` hierarchy mapping
```
Hero       → establishes identity, headline, two clear CTAs
  ↓
Mission    → explains the three-pillar framework (the "why")
  ↓
About      → narrative explanation + quantified urgency (480 ton stat)
  ↓
Projects   → concrete proof of activity (the "what")
  ↓
Volunteer  → conversion section (the "how to help")
  ↓
Impact     → validation and social proof (stats reinforce credibility)
  ↓
Newsletter → low-commitment re-engagement CTA
  ↓
Footer     → navigation + contact (utility layer)
```

This funnel moves from **awareness → understanding → proof → action → trust → re-engagement**, which is the correct conversion architecture for a non-profit homepage.

### Improvements over the current hierarchy
- The hero now has a visible paragraph and two differentiated CTAs ("Learn More" and "Volunteer With Us") instead of just a heading.
- Impact stats are elevated from a buried widget into a dedicated full-width dark section.
- The volunteer section is no longer hidden — it is a prominent grid of four role cards.
- Lorem Ipsum has been replaced with accurate, specific copy throughout.
- The newsletter form appears above the fold of the page's final third, increasing subscription visibility.

## 4. Accessibility Considerations

### Critical issues in the current site (WCAG 2.1 AA)
- **No skip-to-content link** — fails WCAG 2.4.1 (Bypass Blocks). Keyboard users must tab through all navigation items on every page load.
- **All images have empty alt attributes** (`alt=""`) — fails WCAG 1.1.1 (Non-text Content) for informative images. The logo, project images, and about-section image carry visual meaning and need descriptions.
- **Hamburger button has `aria-label="hamburger-icon"`** — the label describes the visual metaphor rather than the action. Should be "Open mobile menu".
- **Counter widgets use JavaScript to animate from 0 to a number** but have no `aria-live` region — screen readers never announce the final value.
- **Colour contrast** — the orange button (`#FF8E00`) on a white background (`#FFFFFF`) has a contrast ratio of approximately 2.6:1 against white, below the 4.5:1 minimum for text (WCAG 1.4.3).
- **No `lang` attribute on HTML element** — set to `en-US` only; the bilingual content contains no `lang` switching.
- **Focus styles** — Elementor strips default browser focus outlines (`outline: none` is applied globally in frontend CSS). There is no custom focus style to replace it.
- **Modal/off-canvas menu** — the mobile menu overlay has no `role="dialog"`, no `aria-modal`, and no focus trap.

### Improvements implemented in `index.html`
- `<a class="skip-link" href="#main-content">` implemented and styled in `style.css` (appears on `:focus`).
- All images have descriptive `alt` text tailored to their context.
- All decorative elements have `aria-hidden="true"`.
- Interactive SVG icons have `aria-hidden="true" focusable="false"` to prevent double-announcement.
- Hamburger has `aria-expanded`, `aria-controls`, and a meaningful `aria-label="Open mobile menu"`.
- Mobile menu has `role="dialog"`, `aria-modal="true"`, and `aria-hidden="true"` (toggled by JS).
- Counter spans have `aria-live="polite"`.
- Newsletter input has an associated `<label>` and `aria-describedby`.
- All external links have `target="_blank" rel="noopener noreferrer"` and `aria-label` text noting the action.
- `style.css` defines a `:focus-visible` outline in teal (3px, offset 2px) applied globally.
- `lang="en"` is correctly set on `<html>`.
- Semantic HTML5 landmark elements used throughout: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`.
- Section headings (`id="heroHeading"`, `id="missionHeading"`, etc.) are linked via `aria-labelledby` on their parent `<section>`.
- Cards use `role="listitem"` within `role="list"` containers for proper list semantics without needing `<ul>/<li>`.

## 5. What the New Design Must Achieve

### Non-negotiable requirements
1. **Load without WordPress/jQuery.** The new site is a static HTML build. Zero dependency on Elementor, ekit, or jQuery. All interactivity is handled by `/js/animations.js` (vanilla JS).
2. **Visual identity parity with the design system.** Every CSS custom property defined in `style.css` must be honoured. No inline colour overrides except where explicitly needed for one-off dark-section text colour.
3. **Mobile-first responsive.** All grids collapse gracefully: 3-col → 1-col (MVV, Projects), 4-col → 2-col → 1-col (Volunteer), 2-col → 1-col (Impact, Hero), 3-col → 1-col (Footer).
4. **Animations on scroll.** AOS library handles section-level fade-up/fade-right/fade-left. The `reveal` class + IntersectionObserver in `animations.js` handles card-level animations. The hero uses keyframe animations defined in `style.css`.
5. **Verbatim content strings preserved.** The following strings must appear exactly as specified and have been verified in `index.html`:
   - H1: "Welcome to Pro Hands: Empowering Communities for a Sustainable Future!" ✓
   - Mission: "Educating communities through creative mediums like puppetry and theatre for environmental awareness." ✓
   - Vision: "A sustainable Jordan led by its own communities, inspiring environmental stewardship and responsibility." ✓
   - Values: "Inclusivity, creativity, sustainability and community engagement — fostering long-term impact." ✓
   - Phone: "+962 7 9510 6049" ✓
   - Email: "ProHands.Training@protraining.org" ✓
   - CTAs: "Learn More", "Volunteer With Us", "Read More", "Apply Now", "Subscribe" ✓

### Design goals for CSS & animation agents
- `visual-3d.css` must define the `.globe-3d` component: a CSS 3D sphere with three orbital rings and an animated rotating core. Colour theme: teal/navy gradient with white orbital lines.
- `animations.css` must define `.particle` animations (random float paths via keyframes), `.card-3d` perspective hover effect, and the hero stagger sequence.
- `animations.js` must initialise: hamburger toggle (with `aria-expanded` updates), mobile menu open/close (with focus trap), sticky navbar scroll class, `IntersectionObserver` for `.reveal` elements, and animated number counters for `.js-counter` elements.
- The language toggle (`#langToggle`) should toggle `dir="rtl"` on `<html>` and swap button text between `عربي` and `English`. RTL rules are already defined in `style.css`.

### Performance targets
- Largest Contentful Paint (LCP): under 2.5 seconds on 4G (logo image and hero h1 are LCP candidates — both have `loading="eager"` and `fetchpriority="high"`).
- No render-blocking resources beyond Google Fonts (acceptable trade-off for brand typography).
- AOS script is deferred by placement at bottom of `<body>`.
- All below-fold images use `loading="lazy"`.
- Total CSS: style.css + visual-3d.css + animations.css should target under 40 KB combined (uncompressed).
